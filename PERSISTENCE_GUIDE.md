## 🔄 Flux de Persistance

### 1️⃣ Créer un Acte (FirebaseService.createActe)

```dart
Future<bool> createActe(ActeNaissance acte) async {
  // 1. Stocker localement (offline-first)
  _actes[acte.id] = acte;
  _allActes.add(acte);
  
  // 2. Stocker dans Firestore collection "actes"
  await FirebaseFirestore.instance
      .collection('actes')
      .doc(acte.id)  // ← acte.id comme document ID
      .set(acte.toJson());
  
  return true;
}
```

**Résultat**:
- ✅ Acte en cache local
- ✅ Acte dans Firestore collection "actes"
- ✅ Statistique dans Firestore collection "actes_stats" (séparé)

---

### 2️⃣ Récupérer un Acte (FirebaseService.getActeById) ⭐ NOUVEAU

```dart
Future<ActeNaissance?> getActeById(String acteId) async {
  // Étape 1: Vérifier cache local (rapide)
  if (_actes.containsKey(acteId)) {
    return _actes[acteId];
  }
  
  // Étape 2: Chercher dans Firestore collection "actes"
  final docSnapshot = await FirebaseFirestore.instance
      .collection('actes')
      .doc(acteId)
      .get();
  
  if (!docSnapshot.exists) {
    return null; // Acte introuvable
  }
  
  // Étape 3: Convertir et mettre en cache
  final acte = ActeNaissance.fromJson(docSnapshot.data()!);
  _actes[acteId] = acte;
  return acte;
}
```

**Logique**:
1. Cache local d'abord (offline)
2. Firestore ensuite (en ligne)
3. Résultat mis en cache
4. `null` si non trouvé

**Avant**:

**Après**:

```dart
void _consultExtract() async {

```

**Bénéfices**:
- ✅ Retrouve l'acte même après redémarrage

### 4️⃣ Écran Vérification (VerificationScreen)

**Avant**:
```dart
Future<void> _verifyActe() async {
  final acte = firebaseService.getActe(acteId);
  // ❌ Cherche uniquement en mémoire
}
```

**Après**:
```dart
Future<void> _verifyActe() async {
  // ✅ Cherche dans Firestore (persistance complète)
  final acte = await firebaseService.getActeById(acteId);
  
  // Vérifier blockchain
  final record = blockchainService.getRecord(acteId);
  final verified = record != null && acte.blockchainHash == record.hash;
}
```

**Bénéfices**:
- ✅ Peut vérifier n'importe quel acte du système
- ✅ Indépendant de la mémoire locale
- ✅ Vérification fiable

## 📊 Scénarios de Persistance

### Scénario 1: Agent enregistre une naissance
```
1. Agent entre données → ActeNaissance créé
2. createActe() appelé
   ├─ Sauvegardé en cache local
   ├─ Sauvegardé dans Firestore "actes"
   └─ Statut enregistré dans "actes_stats"
3. QR code généré (contient ID uniquement)
4. ✅ Acte persistant
```

### Scénario 2: Famille récupère acte après redémarrage
```
1. Utilisateur redémarre l'app
2. Cache local VIDE (la map est réinitialisée)
3. Famille saisit ID → _consultExtract() appelé
4. getActeById() cherche:
   ├─ Cache local: VIDE (première fois)
   └─ Firestore: TROUVÉ ✅
5. Acte chargé et affiché
6. Télécharger PDF possible
```

### Scénario 3: Vérificateur scanne QR après redémarrage
```
1. Utilisateur redémarre l'app
2. Vérificateur scan QR → obtient ID
3. _verifyActe() appelé
4. getActeById() cherche:
   ├─ Cache local: VIDE
   └─ Firestore: TROUVÉ ✅
5. Acte chargé + blockchain vérifié
6. Résultat: "Acte vérifié ✅"
```

### Scénario 4: Pas de connexion Internet (Offline)
```
1. Agent enregistre acte (online)
   ├─ Sauvegardé localement
   └─ Sauvegardé dans Firestore ✅
   
2. Connexion Internet coupée
   
3. Utilisateur cherche acte (offline)
   ├─ Cache local: TROUVÉ (de avant)
   └─ Firestore: NON ACCESSIBLE (offline)
   → Acte retrouvé depuis cache ✅
   
4. Reconnexion: cache invalidé, Firestore utilisé
```

## 🔑 Points Clés d'Implémentation

### 1. Firestore.collection("actes")
- **Document ID**: `id_acte` (ex: `GN-2026-12345`)
- **Données**: Acte complet (via `toJson()`)
- **Pas de données sensibles inutiles**: Seulement ce qui est nécessaire

### 2. Fonction getActeById()
- **Cache-first**: Vérifier local avant Firestore
- **Firestore-second**: Chercher dans Firestore si non trouvé localement
- **Async/await**: Opération asynchrone pour UI réactive
- **Null-safety**: Retourner `null` si non trouvé

### 3. Écrans mettent à jour
- **FamilleScreen**: `_consultExtract()` → `async` + `getActeById()`
- **VerificationScreen**: `_verifyActe()` → `await getActeById()`

### 4. Modèle ActeNaissance
- **toJson()**: Sérialisation complète
- **fromJson()**: Désérialisation complète
- **Tous champs**: Y compris `prefecture` (pour actes_stats)
